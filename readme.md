  추천하는 수정 순서
  
   1. 역기획서 작성 (완료)
   2. 정적 분석 및 CI 기초 설정 (추가/조정)
       * 코드 개발 전, flake8, black, isort 같은 린터와 포매터를 설정하고 GitHub
         Actions에서 이를 검사하도록 먼저 세팅합니다. 이는 팀 전체의 코드 품질을
         상향 평준화합니다.
   3. 코드 개발 & 유닛 테스트 (병행)
       * 별도의 단계보다는 TDD(Test-Driven Development) 방식으로 병행하는 것이
         좋습니다. 유닛 테스트는 코드의 가장 작은 단위를 검증하므로 개발과 동시에
         이루어져야 합니다.
   4. 통합 테스트
       * DB 연동, 외부 API 서비스와의 상호작용 등 구성 요소 간의 연결을 확인합니다.
   5. API 테스트 (순서 상향)
       * UI 테스트보다 먼저 수행해야 합니다. 백엔드 API가 견고해야 이를 사용하는
         프론트엔드(UI) 테스트가 의미를 갖기 때문입니다.
   6. 데이터 검증 (순서 상향)
       * 마이그레이션 테스트 및 DB 스키마 무결성 검증은 기능 테스트 전에 완료되는
         것이 안정적입니다.
   7. 기능 테스트 (Functional Test)
       * 사용자 시나리오(역기획서 기반)에 따라 비즈니스 로직이 의도대로 동작하는지
         확인합니다.
   8. UI 테스트 (E2E)
       * 브라우저 수준에서 사용자의 흐름을 테스트합니다 (Playwright, Selenium 등
         활용).
   9. 보안 테스트 (추가)
       * Django의 check --deploy 명령어나 bandit 등을 이용한 기본적인 보안 취약점
         점검이 필요합니다.
   10. 성능 테스트
       * 시스템이 완성된 후 부하/스트레스 테스트를 통해 병목 지점을 찾습니다.

  ---

  요약된 최종 추천 순서

   1. 역기획서 작성 (완료)
   2. CI 설정 및 정적 분석 (Linter, Formatter, GitHub Actions)
   3. 코드 개발 & 유닛 테스트 (Logic 단위)
   4. 통합 테스트 (Service/DB 연동)
   5. API 테스트 (REST API 엔드포인트 검증)
   6. 데이터 검증 (Migration, DB Integrity)
   7. 기능 테스트 (Business Requirement 기반)
   8. UI 테스트 (User Journey, E2E)
   9. 보안 테스트 (Security Scan - 신규 제안)
   10. 성능 테스트 (Load/Stress Test)
